### 데이터 흐름 요약

1. **디바이스**: 카메라 → YOLO → ROI 판정 → 릴레이 + 음성 송출
2. **디바이스 → 대시보드**: 탐지 이벤트마다 `POST /events/ingest` (장치 API Key 인증)
3. **대시보드 → 디바이스**: ROI 설정/음성 매핑 변경 시 디바이스가 주기적 polling(60초)으로 `GET /devices/{me}/config` 호출하여 자체 갱신
4. **영상**: 디바이스의 MJPEG 서버(`:8080/stream.mjpg`)를 대시보드가 프록시 → 브라우저 `<img src="/api/stream/{id}.mjpg">`
5. **실시간 알림**: 디바이스 이벤트 → 백엔드 → WebSocket → 브라우저

## 11. 범례

| 항목 | 설명 |
|------|------|
| **디바이스 ID 체계** | `DV-{HW\|SW\|AI}-NNN` (HW=하드웨어, SW=소프트웨어, AI=AI/모델) |
| **대시보드 ID 체계** | `DB-{UI\|API\|SVC\|MDL\|CMN\|WS}-NNN` (UI=화면, API=라우터, SVC=서비스, MDL=DB 모델, CMN=공통, WS=WebSocket) |
| **계층 색상** | 노랑=HW/Service · 초록=SW/API · 주황=AI/Model · 파랑=UI · 회색=Common · 보라=WebSocket |
| **인증 표기** | 관리자 = JWT 인증 / 디바이스 = X-API-Key 헤더 / 공개 = 인증 불필요 |
| **우선순위** | 상 = MVP 필수 / 중 = 1차 릴리스 / 하 = 2차 릴리스 |
| **공수 단위** | 1 = 1 man-day (8시간) |
| **타입 표기** | Pydantic(백엔드) 또는 TypeScript(프론트) 타입 그대로. `?`는 선택 |
| **예외 처리** | HTTP 상태 코드 + 사용자 메시지. 4xx는 사용자 노출, 5xx는 마스킹 후 로그 |
| **좌표 체계** | ROI 폴리곤은 영상 크기에 무관한 정규화 좌표(0~1) 사용. 렌더 시 실제 해상도 곱함 |
| **시간/타임존** | DB는 UTC / API 응답은 ISO 8601 / UI는 Asia/Seoul 변환 |
| **WebSocket** | JWT는 쿼리. close 1000=정상, 1008=인증실패, 1011=서버오류. 메시지: `{ type, payload }` |
| **디바이스 인증** | 디바이스 등록 시 1회만 평문 API Key 노출. 이후 sha256 해시만 DB에 저장. 분실 시 재발급(`/regenerate-key`) |
